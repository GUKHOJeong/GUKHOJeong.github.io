2. Challenge (Situation & Task)

üßê Situation

Users struggle to utilize products because they don't read voluminous PDF manuals, and companies face the problem of increased customer service costs due to simple repetitive inquiries. An AI chatbot service capable of understanding both text and images was needed to solve this.

üöß Task

The key task was to build a high-performance RAG engine that grasps context rather than simple keyword search, and to design a backend architecture that guarantees real-time performance.

‚ö†Ô∏è Difficulties Faced:

We had to overcome the significantly low search accuracy when initially integrating LangChain and FAISS, and the limitation where context was not maintained due to single-turn QA processing.

3. Process & Action

üîç RAG Quality Enhancement: 2-Stage Retrieval

  • Problem Analysis: Discovered that valid documents were missing from search results due to FAISS's post-filtering method.
  • Ablation Study: Experimented with fetch-k values (10~500), MMR, and cosine similarity thresholds, confirming that valid documents were not retrieved when fetch-k was less than 100.
  • Solution: We drastically improved accuracy by building a 2-stage pipeline: increasing fetch-k to 500 to broaden document retrieval (Recall boost), and then introducing a Re-ranker to reorder by relevance.

ü§ñ Multi-Tool Agent & Session Memory

  • Feature Expansion: Applied a Multi-Tool structure to the Agent to enable 'Product Recommendation' and 'Comparative Analysis' beyond simple answers.
  • Context Maintenance: To overcome the limits of one-off Q&A, we implemented logic to inject Session Memory into the Agent, enabling continuous conversation that understands the context of the user's previous questions.

‚ö° Real-time Multimodal System Engineering

We established a real-time bi-directional communication environment with the frontend by applying WebSocket on top of FastAPI.

  • Multimodal Processing: Processed images within retrieved information via LangGraph and dynamically inserted image links into the Context to provide intuitive answers combining text and images.
  • Security: Implemented a secure user management system using JWT authentication and password hashing.

4. Result

üìà Key Achievements

  • Service Launch: Successfully launched the MVP service through the complete integration of frontend and backend logic.
  • Technical Significance: Overcame the limitations of simple vector search with Re-ranking techniques to secure reliable search quality, and proved the possibility of multimodal services providing visual information beyond text-centric LLM limits.

5. Limitations & To-Be Roadmap

‚ö†Ô∏è Unresolved Issue: Context Window Limit

Context Truncation: Due to current LLM Context Window limits, the problem of truncated context or missing information still exists when conversations get long or referenced documents are numerous. We plan to solve this by introducing Long-Context models or advancing Memory Summarization algorithms.

‚öñÔ∏è Insight: Response Speed vs UX (Latency)

Physical latency due to the RAG process exists, but we launched the MVP judging it to be serviceable compared to the average response times of commercial models like GPT-4 or Gemini. We plan to fully introduce Token Streaming to make characters appear immediately as the answer is generated, performing visual optimization to reduce Perceived Latency.
