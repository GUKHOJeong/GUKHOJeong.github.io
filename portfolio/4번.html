2. 배경 및 문제 정의 (Challenge)

🧐 서비스 목표

현대인은 수많은 뉴스에 노출되지만 정작 중요한 맥락을 파악하기 어렵습니다. 특정 키워드에 대한 최신 뉴스를 실시간으로 수집하고, 내용을 3줄로 요약하며, 주요 토픽별로 묶어주는 서비스를 기획했습니다.

🚧 주요 난제 (Difficulties)

  • 중복 뉴스 범람: 속보 경쟁으로 인해 같은 내용의 기사가 여러 언론사에서 3~4개씩 반복 생산되어 분석 품질을 저하시킴.
  • 요약 모델의 일관성: KoBERT 등 초기 모델 사용 시, 긴 문장에 대해 맥락을 잃거나 문법적으로 어색한 요약문이 생성됨.
  • 응답 지연(Latency): 자연어 처리(요약+토픽모델링) 과정이 무거워 사용자 대기 시간이 길어지는 문제.

3. 해결 과정 (Process & Action)

🧠 NLP 파이프라인 구축

  1. 모델 선정 (KoBERT vs T5):
    - KoBERT: 긴 입력 처리에 강점은 있으나 생성 결과가 불안정.
    - T5-Base (Selected): Prompt 기반("요약: ...") 미세 조정을 통해 가장 자연스러운 한국어 생성 능력을 보여 선정함.
  2. 토픽 모델링 (LDA): Mecab 형태소 분석기로 명사만 추출한 후, Scikit-learn의 LDA를 적용해 뉴스를 주제별로 자동 분류.

🔧 트러블슈팅: 데이터 품질 및 UX 개선

  • 중복 기사 제거: 언론사별 그룹핑 및 TF-IDF 기반 코사인 유사도를 계산하여, 내용이 80% 이상 일치하는 '복사 붙여넣기'성 기사를 필터링.
  • 실시간 피드백(UX): 긴 분석 시간을 고려하여 `Flask-SocketIO`를 도입, "현재 데이터 수집 중...", "요약 모델 구동 중..."과 같은 진행 상황을 프로그레스 바로 시각화해 이탈률 감소 유도.
