Key Features

  • Real-time News Data Collection: Real-time crawling of Naver News data based on the keywords and time period entered by the user.
  • AI-Powered Automatic Summarization: Utilizing a summarization model from the Transformers library to automatically summarize the core content of each news article.
  • Topic Modeling and Classification: Automatically classifying collected news articles into major topics using Mecab and Scikit-learn's LDA model.
  • Data Visualization: Automatically generating a WordCloud to visually represent the core keywords for each topic.
  • Real-time Processing Visualization: Providing real-time feedback to users by visualizing the entire process from data collection to analysis with a progress bar.

Models Attempted

  • KoBert-based summarization model
  • T5-small model
  • T5-base model

Final Model: T5-base Model

The content was extracted from crawled data using a model from Hugging Face, followed by fine-tuning on eenzeenee/t5-base-korean-summarization.


Model Address (Model Used)



Reasons for Final Model Selection

  • KoBert: Although it had the advantage of handling long input tokens, the actual summary performance was somewhat unstable.
  • T5: The T5 model required a prefix like "summarize:" as an input constraint but generated more consistent and higher-quality summaries compared to KoBERT.
  • T5-small vs. T5-base: The small model showed high performance compared to KoBERT, but given the sufficient resources and the base model's superior generalization ability, T5-base was chosen as the final model.

Problem

Data duplication occurred because the same reporter would often write 3-4 news articles with similar content.


Solution

A solution is planned to be implemented to remove duplicate news articles by calculating the similarity between them after grouping by reporter or news agency.


Final Insights

  • A significant amount of time was spent on collecting and analyzing a large volume of news, leading to a long wait time for users.
  • However, a plan is in place to connect the service to a database so that users can instantly view their previous search records, improving the user experience.
