2. 배경 및 문제 정의 (Challenge)

🧐 배경 (Situation)

사용자는 방대한 PDF 매뉴얼을 읽지 않아 제품 활용에 어려움을 겪고, 기업은 단순 반복 문의로 인한 고객센터 비용 증가 문제를 안고 있었습니다. 이를 해결하기 위해 텍스트와 이미지를 모두 이해하는 AI 챗봇 서비스가 필요했습니다.

🚧 과제 (Task)

단순 키워드 검색이 아닌 문맥을 파악하는 고성능 RAG 엔진을 구축하고, 실시간성이 보장되는 백엔드 아키텍처를 설계하는 것이 핵심 과제였습니다.

⚠️ 직면한 난관:

초기 LangChain과 FAISS 연동 시, 검색 정확도가 현저히 떨어지는 문제와 단발성 질문 처리로 인해 문맥이 유지되지 않는 한계를 극복해야 했습니다.

3. 해결 과정 (Process & Action)

🔍 RAG 검색 품질 고도화: 2-Stage Retrieval 도입

  • 문제 분석: FAISS의 후-필터링(Post-filtering) 방식으로 인해 실제 유효한 문서가 검색 결과에서 누락되는 현상을 발견했습니다.
  • 가설 검증 (Ablation Study): fetch-k 값(10~500), MMR, 코사인 유사도 임계값을 조절하며 실험한 결과, fetch-k가 100 미만일 때 유효 문서가 검색되지 않음을 확인했습니다.
  • 해결책: fetch-k를 500까지 늘려 폭넓게 문서를 가져온 뒤(Recall 증대), Re-ranker(리랭커)를 도입하여 관련성 높은 순서로 재정렬하는 2단계 파이프라인을 구축해 정확도를 비약적으로 향상시켰습니다.

🤖 Multi-Tool Agent 및 세션 메모리 구축

  • 기능 확장: 단순 답변을 넘어 '상품 추천', '비교 분석' 등이 가능하도록 Agent에 Multi-Tool 구조를 적용했습니다.
  • 문맥 유지: 일회성 문답의 한계를 넘기 위해 Agent 내에 과거 대화 기억(Session Memory)을 주입하는 로직을 구현하여, 사용자의 이전 질문 맥락을 이해하는 연속적인 대화가 가능하도록 했습니다.

⚡ 실시간 멀티모달 시스템 엔지니어링

FastAPI 기반에 WebSocket을 적용하여 프론트엔드와 실시간 양방향 통신 환경을 구축했습니다.

  • 멀티모달 처리: 검색된 정보 내 이미지를 LangGraph를 통해 처리하고, Context 내에 이미지 링크를 동적으로 삽입하여 텍스트와 이미지가 결합된 직관적인 답변을 제공했습니다.
  • 보안: JWT 인증 및 비밀번호 해시(Hashing) 적용으로 안전한 사용자 관리 시스템을 구현했습니다.

4. 결과 및 성과 (Result)

📈 주요 성과

  • 서비스 런칭: 프론트엔드와 백엔드 로직의 완전한 통합을 통해 MVP 서비스를 성공적으로 런칭했습니다.
  • 기술적 의의: 단순 벡터 검색의 한계를 리랭킹(Re-ranking) 기법으로 극복하여 신뢰할 수 있는 검색 품질을 확보했으며, 텍스트 중심의 LLM 한계를 넘어 시각 정보까지 제공하는 멀티모달 서비스의 가능성을 입증했습니다.

5. 한계점 및 향후 발전 방향 (Limitations & To-Be Roadmap)

⚠️ Unresolved Issue: Context Window 한계

컨텍스트 잘림(Context Truncation) 현상: 현재 LLM의 Context Window 제한으로 인해, 대화가 길어지거나 참조 문서가 많을 경우 이전 대화의 맥락이 잘리거나 정보가 누락되는 문제가 여전히 존재합니다. 향후에는 Long-Context 모델 도입 또는 기억 요약(Memory Summarization) 알고리즘 고도화를 통해 이를 해결할 계획입니다.

⚖️ Insight: 응답 속도와 사용자 경험 (Latency & UX)

RAG 프로세스로 인한 물리적 응답 지연이 존재하지만, GPT-4나 Gemini 등 상용 모델의 평균 응답 시간과 비교했을 때 서비스 가능한 수준으로 판단하여 MVP를 런칭했습니다. 추후에는 Token Streaming (스트리밍 모드)를 본격적으로 도입하여, 답변이 생성되는 즉시 화면에 글자가 나타나게 함으로써 체감 대기 시간(Perceived Latency)을 단축시키는 시각적 최적화를 진행할 예정입니다.

🚀 향후 발전 방향 (To-Be Roadmap)

1. 데이터 선순환을 통한 AICC 자동화 (Data Flywheel)
  • Self-Evolving FAQ: 상담 로그를 군집화(Clustering)하여 자주 묻는 질문을 분석하고, 관리자 개입 없이 FAQ 데이터베이스를 자동 업데이트하는 파이프라인 구축.
  • 지식 자산화 (Knowledge Assetization): 휘발성 상담 로그를 기업의 핵심 지식 자산(Knowledge Base)으로 전환하여 상담 운영 효율 극대화.
2. 산업 및 글로벌 확장성 (Global B2B SaaS)
  • Domain Agnostic 확장: 가전뿐만 아니라 의료기기, 자동차, 조립 가구 등 설명서가 있는 모든 산업군에 즉시 적용 가능한 솔루션화.
  • Global Cross-Lingual Search: 한국어 매뉴얼로 영어/일본어 질문에 응답하는 다국어 임베딩 및 실시간 번역 파이프라인 구축.
  • 디지털 포용성: 복잡한 공공/행정 문서를 대화형으로 변환하여 누구나 쉽게 정보에 접근할 수 있도록 개선.
